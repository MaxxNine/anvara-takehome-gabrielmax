# Homepage Sections Proposals

## Direction

This file is a focused section brief for the homepage direction we should discuss next.

The visual system should be:

- bright-white overall
- cinematic, but still bright
- product-led, not illustration-led
- cobalt blue as the main action color
- pale blue surfaces for depth
- very small lime accents for data moments only

Reference mix:

- original Anvara homepage: product-preview hero, strong pacing, immediate proof
- `references/lp1.jpg`: white canvas, cobalt emphasis, soft blue glow
- `references/lp2.jpg`: strong composition, occasional ink blocks, lime only as a utility accent

Structural rule:

- the hero owns the interactive inventory preview
- later sections should not repeat the same tabbed-card pattern
- each section must earn its place with a distinct role

- Background: `#F7F8FC`

- Soft Accent: `#9BB7FF`
- Highlight Tint: `#DCE7FF`
- Data Accent: `#D9FF3F`
- Border: `#E7EBF3`
- Ink Card: `#111827`

Usage rules:

- Keep the hero light, high-exposure, and airy.
- Use cobalt for CTAs, active tabs, and highlighted words.
- Use pale blue for secondary surfaces and glow treatments.
- Use lime only for charts, badges, or tiny metric highlights.
- Avoid large dark backgrounds and avoid full dark-hero treatment.

## 1. Hero

### Proposal

A bright cinematic hero with:

- large headline
- short supporting copy
- primary CTA
- secondary CTA
- interactive format explorer

### Description

This is still the most important section. The hero should preview the product in motion, but without copying the original Anvara search interaction. The right pattern is a clean format switcher with `Display`, `Video`, `Podcast`, and `Newsletter` tabs that reveal preview cards with publisher, price, type, and availability.

### Visual notes

- white or pale-blue hero field
- soft glow behind the media area
- lightly frosted or glassy cards
- bright cinematic video only if it stays luminous
- no dark overlay-heavy hero

---

## 2. Trust + Breadth Band

### Proposal

A compact proof band directly below the hero that also communicates marketplace breadth.

### Description

This replaces both the logo-strip function and the standalone rotating-category section. Since we do not want fake brand logos, this band should communicate marketplace density and category breadth in one move.

Suggested content:

- `20+ placements across 4 formats`
- `Verified publishers`
- `Transparent monthly pricing`
- `Browse active inventory`
- `Display, video, podcast, and newsletter placements`

### Visual notes

- thin bordered row
- white background
- compact icon or chip system
- subtle cobalt dividers
- one stronger breadth statement can sit inside the band as a headline or subheadline, rather than becoming its own section

---

## 3. Featured Inventory

### Proposal

A featured-inventory section with one dominant card and a few supporting cards.

### Description

This section should not duplicate the hero interaction. The hero already previews formats interactively. This section should instead feel more editorial and curated, almost like a “featured placements” window into the marketplace.

It should still show real inventory structure:

- slot name
- publisher
- short description
- type
- price
- availability

### Visual notes

- pale-blue section with one large anchor card
- two or three smaller supporting cards
- rounded corners
- soft shadow and clean borders
- more curated than grid-like
- avoid a repeated 4 to 6 equal-card layout

---

## 4. Sponsor / Publisher Split

### Proposal

Two large cards:

- For Sponsors
- For Publishers

### Description

This should appear only after the product is already clear. The user first understands the marketplace, then sees how it serves their role. Each card should explain the value in 3 short points, not heavy paragraphs.

### Visual notes

- same structure, different accent treatment
- sponsor card can use cobalt + ink details
- publisher card can use pale blue + white
- keep both cards equally premium

---

## 5. How It Works

### Proposal

A simple 3-step flow:

1. Browse inventory
2. Match placements
3. Book and manage sponsorships

### Description

This section exists to reduce perceived complexity. It should make the product feel straightforward, not operationally heavy.

### Visual notes

- three horizontal cards on desktop
- stacked flow on mobile
- numbered pills or circles
- thin connector line

---

## 6. Alternating Feature Stories

### Proposal

Two or three alternating sections with text + styled product visuals.

### Description

This is how we keep the scroll rhythm strong, like the original reference, without copying its exact art direction. Each feature block should focus on one core promise:

- transparent pricing and availability
- multi-format discovery
- faster sponsor-publisher matching

### Visual notes

- alternate text-left / visual-right, then reverse
- use white and pale-blue sections interchangeably
- allow one single ink-card moment for the most important feature only
- avoid generic dashboard screenshots

### Placement rule

If we use the ink-card treatment from `lp2.jpg`, it should appear once, as the visual anchor for the single most important feature. It should not become a repeating motif across the page, because it will overpower the bright system.

---

## 7. FAQ

### Proposal

A compact accordion near the bottom.

### Description

This is a trust and completeness section. It should answer the obvious practical questions without becoming long-form documentation.

Suggested topics:

- `Can I book a slot directly through Anvara?`
- `How is pricing determined?`
- `What placement formats are available?`
- `What happens if a slot is already booked?`
- `How do publishers list inventory?`

### Visual notes

- simple borders
- spacious padding
- strong type hierarchy
- almost no decoration

---

## 8. Final CTA

### Proposal

A bright closing CTA block with:

- strong line of copy
- small inventory preview cluster
- primary CTA
- secondary CTA

### Description

The end of the page should feel like a refined continuation of the hero, not a tonal switch. Since we are intentionally avoiding dark hero direction, the closing section should also stay bright and premium.

### Visual notes

- white or pale-blue block
- stronger cobalt emphasis
- subtle blur/glow behind preview cards
- concise copy, not another long explanation

---

## Section Rhythm

Recommended pacing:

1. Hero
2. Trust + breadth band
3. Featured inventory
4. Sponsor / publisher split
5. How it works
6. Alternating feature stories
7. FAQ
8. Final CTA

Why this order works:

- product first
- proof and breadth second
- audience split later
- action repeated at the end

This keeps the page readable, persuasive, and easy to build in stages.
